Extensible Markup Modules - (an XML-based) replacement of both HTML and LaTeX, with a Turing-complete syntax.

This is an unofficial proposal.

See ReSpec's user guide for how to get started!

About

See the wiki.

Project goals:

make an "extension" of XHTML that is:

  • based on namespaces
  • feature-rich
  • extensible by anyone (only need to know programming)
  • will support macroses and be Turing-complete
  • replaces both HTML and LaTeX as legacy

As first examples, I made namespaces for Table of Contents and for syntax coloring sources.

The software for this kind of XML processing already exists.

It should be standardized, because the aforementioned software is slow and we need alternative software, too.
